## Фаза 0 — Гигиена (неделя 1)

### T-01 · Merge ветки rank-position
- **Приоритет:** P0 · **Оценка:** 0.5 дн
- Смержить `worktree-blogger-rank-position` в `master` (живой ранг-позиция в Лиге уже готова), разрешить конфликты, удалить worktree.

- **Приоритет:** P1 · **Оценка:** 1 дн

### T-03 · Sentry + мониторинг
- **Приоритет:** P1 · **Оценка:** 1 дн
- Sentry на frontend и edge-функции, Supabase log drains, алерты на ошибки платежей.

- **Приоритет:** P2 · **Оценка:** 2 дн
- `lib/api.ts` (1492 строки) → модули `auth / influencers / campaigns / deals / billing`. Механический рефакторинг, юнит-тесты уже есть.

## Фаза 1 — Production-ready ядро (недели 2–4)

Цель: реальный бренд проходит путь регистрация → подписка → кампания → сделка → чат на живом Supabase.

### T-05 · Payme Merchant API
- **Приоритет:** P0 · **Оценка:** 5 дн
- Сейчас ветка `provider === "payme"` — заглушка. Реализовать edge-функцию `payme-webhook` (verify_jwt=false): `CheckPerformTransaction / CreateTransaction / PerformTransaction / CancelTransaction`. Тарифы в сумах в `plans.ts`.
- ⚠️ Регистрация мерчант-аккаунта — самый долгий внешний блокер, подать заявку немедленно.

### T-06 · Click — второй провайдер
- **Приоритет:** P1 · **Оценка:** 3 дн · **Зависит от:** T-05 (паттерн)
- Тот же webhook-паттерн, что и Payme.

### T-07 · Realtime-чат
- **Приоритет:** P0 · **Оценка:** 2 дн
- `useMessages` без Supabase Realtime-подписки — чат живой только в mock-режиме. Подписка на `postgres_changes` по `deal_id`; публикация уже создана в `004_realtime.sql`.

### T-08 · Realtime-нотификации
- **Приоритет:** P1 · **Оценка:** 1 дн · **Зависит от:** T-07
- `useNotifications` на Realtime — колокольчик без перезагрузки.

### T-09 · Русская локаль
- **Приоритет:** P1 · **Оценка:** 1 дн
- Третий словарь `ru` в `lib/i18n.ts` (~700 ключей). Рынок CIS, а языков только en/uz.

### T-10 · E2E-смоук Playwright
- **Приоритет:** P1 · **Оценка:** 3 дн
- 5 сценариев: регистрация бренда, подписка, создание кампании, бид инфлюенсера, чат. Гонять в CI против preview-деплоя (Vercel + Supabase branch).

### T-11 · Прод-чеклист
- **Приоритет:** P1 · **Оценка:** 2 дн
- Rate limiting на edge-функции, PITR-бэкапы, домен, аналитика (PostHog/Plausible), политика хранения PII в `influencer_contacts`.

## Фаза 2 — Монетизация (недели 5–8)

Цель: комиссия 10–15% со сделок вместо подписки $29. Метрика: первая сделка через эскроу.

### T-12 · Схема эскроу
- **Приоритет:** P0 · **Оценка:** 3 дн
- Таблицы `deal_payments`, `payouts` + RLS-политики + миграция.

### T-13 · Статусная машина сделки
- **Приоритет:** P0 · **Оценка:** 3 дн · **Зависит от:** T-12
- Расширить статусы deal: `funded → in_progress → delivered → released / disputed`. UI-статусы в `DealStatusBadge`, переходы в api-слое.

### T-14 · Оплата сделки на платформе
- **Приоритет:** P0 · **Оценка:** 4 дн · **Зависит от:** T-05, T-13
- Бренд оплачивает сделку через Payme/Stripe, средства held до подтверждения, комиссия платформы 10–15%.

### T-15 · Очередь выплат в админке
- **Приоритет:** P1 · **Оценка:** 2 дн · **Зависит от:** T-14
- Ручные выплаты инфлюенсерам через существующую админ-панель Payments; автоматизация позже.

### T-16 · Freemium + годовые планы
- **Приоритет:** P2 · **Оценка:** 2 дн
- Бесплатный план бренда: 3 открытых контакта/мес. Годовые планы со скидкой.

### T-17 · Boost-аукцион в Лиге
- **Приоритет:** P2 · **Оценка:** 3 дн
- Расширить дневной `influencer_feature` до аукциона позиций в Лиге.
