**Сильное.** Архитектура здоровая: чистый монорепо (frontend / backend / docs), единый data-layer `lib/api.ts` с переключателем mock ↔ Supabase, RLS-политики, 17 миграций с дисциплиной, CI (prettier → eslint → tsc → vitest → build), автообновление данных блогеров (pg_cron + Apify + YouTube + Docker-воркер), security headers, Lighthouse-проходы. Это уровень выше среднего pre-seed проекта.

## Фаза 0 — Гигиена (3–5 дней)

- Смержить `worktree-blogger-rank-position` в master.
- Убрать прямые обращения к `mockDB` из страниц (GodMode, где ещё есть) — всё через `api.ts`; в GodMode добавить Supabase-ветку или спрятать в mock-only.

- Разбить `api.ts` на модули (auth / influencers / campaigns / deals / billing) — механический рефакторинг, тесты уже есть.

## Фаза 1 — Production-ready ядро (2–3 недели)

Цель: реальный бренд может пройти путь регистрация → подписка → кампания → сделка → чат на живом Supabase.

1. **Payme + Click (Uzbekistan).** Реализовать Merchant API Payme: endpoints `CheckPerformTransaction / CreateTransaction / PerformTransaction / CancelTransaction` как edge-функцию `payme-webhook` (verify_jwt=false, как stripe-webhook). Click — вторым провайдером тем же паттерном. Тарифы в сумах, конверсия курса в `plans.ts`.
2. **Realtime-чат и нотификации на проде.** `useMessages` / `useNotifications`: подписка на `postgres_changes` канал по deal_id (публикация уже есть в 004_realtime.sql — использовать её).
3. **ru-локаль** в i18n.ts (третий словарь, ~700 ключей — полдня с LLM + вычитка).
4. **E2E-смоук на Playwright**: 5 сценариев (регистрация бренда, подписка mock, создание кампании, бид инфлюенсера, чат). Гонять в CI против preview-деплоя Vercel + Supabase branch.
5. **Прод-чеклист:** rate limiting на edge-функции, бэкапы (PITR), домен + аналитика (PostHog/Plausible), политика хранения PII в influencer_contacts.

## Фаза 2 — Монетизация по-настоящему (3–4 недели)

Подписка $29 — не бизнес. Деньги в транзакциях.

1. **Эскроу-сделки.** Бренд оплачивает сделку через платформу (Payme/Stripe), средства помечаются held; после подтверждения выполнения — выплата инфлюенсеру минус комиссия 10–15%. Таблицы `deal_payments`, `payouts`; статусная машина deal расширяется (`funded → in_progress → delivered → released / disputed`).
2. **Выплаты:** старт вручную (админ-панель Payments уже есть — добавить очередь выплат), автоматизация позже.
3. **Ценовые эксперименты:** freemium для брендов (3 открытых контакта/мес), годовые планы, boost-слоты в Лиге (у influencer_feature уже есть день-кадэнс — расширить до аукциона позиций).

## Фаза 3 — Рост и defensibility (1–2 месяца, параллельно)

1. **Данные как ров.** Скрейпинг-пайплайн уже лучший актив. Добавить: историю метрик (analytics_history уже есть — строить тренды 90 дней), fake-follower score, прогноз ER. Публичная Лига = SEO-магнит: SSR/prerender публичных страниц блогеров (мигрировать public-роуты на пререндер через vite-plugin или отдельный Astro-сайт) — сейчас SPA не индексируется толком.
2. **AI-матчинг** кампания → шорт-лист блогеров (embedding по категории/аудитории/цене) — дешёвая фича, сильная демка для инвесторов.
3. **Telegram-бот** для инфлюенсеров (нотификации о бидах/сделках) — в UZ Telegram главный канал, дешевле мобильного приложения.
4. **Расширение CIS:** KZ как второй рынок (скрейпер уже умеет REGIONS=UZ,KZ,RU), локальные платежи Kaspi — фаза 4.

## Метрики успеха по фазам

- Ф1: 1 живой платёж Payme; чат работает на проде; E2E зелёный.
- Ф2: первая сделка с комиссией через эскроу; take rate ≥ 10%.
- Ф3: 1000+ органических визитов/мес на публичные профили; 20% сделок из AI-матчинга.
